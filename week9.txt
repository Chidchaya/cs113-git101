การ clone git repository งานเก่าที่เคยทำค้างไว้ มาทำงานต่อ โดดยการคะดลอก url งานเดิม มาใส่ใน  git bash 
แล้ว พิมพ์คำว่า git clone ต่อด้วย url ของงานเรา ใน github
การสร้าง หรือ แยก branch สิ่งแรกคือ clone งานตัวเก่ามาก่อน แล้วก็ cd เพื่อเปลี่ยน working directory 
หลังจากนั้น ให้ใช้คำสั่ง git branch ตามด้วยชื่อ branch ที่เราอยากตั้ง
คำสั่ง git branch คือเอาไว้เช็ค ว่า มีbranch อะไรบ้าง ถ้ามี สัญลักษณ์ * นี้ขึ้นมานั่นคือ เป็นbranch ที่กำลังทำงานอยู่ 
git checkout ตามด้วยชื่อ branch คือการเปลี่ยน branch ไปที่ไฟล์นั้น 
การ merge งานเข้าด้วยกัน ให้เข้าไปอยู่ในไฟล์งานไฟล์เดียว ก็คือเราต้อง checkout ไปอยู่ในไฟล์ที่เราต้องการเอาข้อมูลไปรวมหรือ merge แล้วก็ใช้คำสั่ง git merge 
พอ merge เสร็จแล้ว จะเห็นว่า มีชื่อิไฟล์ที่้เรา merge เข้าไปอยู่ในไฟล์ใหญ่ที่เป็นไฟล์รวมงานของเรา มันเหมือนการแทรกข้อมูลเข้าไปในไฟล์ ประมาณนั้น 
ใช้คำสั่ง  git log adog แล้วจะเห็นเส้นของการรวม branch อยู่ ทำให้เรารู้เส้นทางของการmerge ของเรา ว่ามันเข้าไปทางไหน 
เสร็จแล้ว ก็  git push origin master เพื่อ อัพไฟล์งานขึ้น github 
การ pull แบบไม่มี conflict เหมือนเรามีอีกคนมาแก้งานเราโดยเราไม่รู้ตัว โดยที่อีกคนก็สร้างไฟลที่คนละชื่อกับของเรา
ส้รางขึ้นมาแก้ไขงานเรา หรือพัฒนางานเราต่อ โดยที่เราไม่รู้ตัว พอกลับมาเข้าดูอีกที 
ก็จะมีชื่อไฟล์อีกไฟล์หนึ่ง จากอีกคน ขึ้นมาด้วย และเนื้อหางานเราก็อาจจะถูกเปลี่ยนแปลงไป ตามที่อีกคนเปลี่ยน
